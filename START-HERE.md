# 🚀 START HERE — Community Automation Hub, финальный пакет

Это единая точка входа. Ниже — что в пакете, зачем каждый файл и в каком порядке действовать. Всё остальное можно открывать по ссылкам отсюда, не читая всё сразу.

---

## Что это

Система для BrnoWalkers + Družina Moravy, где **одна строка в Google Sheets** запускает всё остальное: создание события в календаре, анонс в Telegram, живую карточку на сайте, приём регистрации через Tally с вейверами под конкретный тип мероприятия, приём оплаты через Revolut, напоминание за день, и автоматическую анонимизацию персональных данных по истечении срока хранения.

**Стоимость: $0 навсегда.** Никакого Make.com, никаких платных тарифов — только Google Apps Script (бесплатная квота личного аккаунта), Tally Free, Telegram Bot API, GitHub Pages, Revolut.

**Единый источник правды:** лист `Events` в Google Sheets. Всё остальное — календарь, Telegram, сайт — production reflects этот лист, никогда не редактируется отдельно.

## Порядок действий с нуля

Это самая короткая версия. Всё то же самое подробно — в `setup-checklist.md`.

1. **Сайт:** пока не трогай — вернёшься к нему на шаге 9.
2. **Google Sheets:** импортируй `events-registrations-template.xlsx`
3. **Telegram-бот:** через `@BotFather`, добавь в канал
4. **Google Calendar:** создай публичный клубный календарь
5. **Tally:** создай форму по `tally-form-structure.md` (это самый долгий шаг — там 10 категорий условной логики)
6. **Apps Script:** вставь `Code.gs` + `appsscript.json`, заполни CONFIG, поставь 3 триггера, задеплой Web App
7. **Свяжи:** Tally webhook → Web App URL; Web App URL → `EVENTS_API_URL` в `index.html`
8. **Протестируй:** тестовое событие сквозь весь цикл (детали — `setup-checklist.md`, пункты 12–16)
9. **Сайт:** теперь залей `druzhina-site-FINAL.zip` в GitHub — карточки и календарь заработают сразу же, т.к. `EVENTS_API_URL` уже вставлен на шаге 7

## Что физически нельзя сделать за тебя

- Создание самого Telegram-бота, Google Calendar, Tally-формы — это разовые действия в твоих аккаунтах
- Коммит `index.html` в GitHub — нет доступа на запись в репозиторий, в реестре MCP-коннекторов подходящего не нашлось
- Заявка на Meta App Review для автопостинга в Facebook — требует верификации юрлица документами от тебя лично

## Важное ограничение, которое стоит держать в голове

**Описания событий на сайте одноязычны** — то, что ты напишешь в Title/Description в Sheets, отображается ровно в этом языке независимо от переключателя RU/EN/CS/UK на сайте. Это осознанный компромисс, а не недоработка: настоящая мультиязычность означала бы писать 4 версии текста на каждое из ~30 событий/месяц, что противоречит самой цели "одна строка — ноль лишней работы". Бейдж клуба, дата и цена — переведены полностью автоматически, это не текст, а конечные значения.

## Что не входит в этот пакет (и почему)

Ранее в разговоре обсуждалась гибридная версия с Make.com — тот файл (`community-hub-v2-scheme.md`) сюда **не включён**. Причина прямая: он описывает другую архитектуру (Make.com как оркестратор вместо чистого Apps Script), которую мы явно отклонили в пользу гарантированного $0 навсегда. Включать его в финальный пакет означало бы нарушить тот самый принцип "один источник правды", который мы строили — два противоречащих друг другу описания системы это ровно то, от чего мы уходим на уровне архитектуры, и это же правило применимо к документации.

По той же причине не включён более ранний `gdpr-consent-text-4lang.md` — в нём было использовано неверное правовое основание (согласие вместо исполнения договора). Актуальный текст — внутри `tally-form-structure.md`.
